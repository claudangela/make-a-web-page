Summary of Concepts

The Basics of Programming: Web, HTML and CSS

Programming

Programming is the process of developing and implementing various sets of instructions to enable a computer to do a certain task. The most important thing about its definition is to understand the types of programming thinking, as follows:

  • Procedural thinking: clear and unambiguous instructions for a computer to follow;
  • Abstract thinking: finding a generality amongst seemingy different things;
  • Systems thinking: break a big problem into smaller pieces;
  • Technological empathy: undestand how computers works; and
  • Debugging: relentless identifying the cause of a computer program not to work. (Debugging process: collect evidence > generate theories > test theories > fix problem)

Web

The World Wide Web, or just the WEB is a collection of (HTML) documents linked among each other. It comprises of 4 major pieces:

  • The Browser: a program that runs on your computer to display files found on the web;
  • The Internet: the world’s largest computer network;
  • The HTTP: the main protocol of the web; and
  • The Servers: computers that host the files that make up the web.

HTML

Or Hypertext Markup Language controls the structure of a page. It is the standard markup language used to create web pages. Its basic vocabulary includes:

  • Tag: is always contained within angled brackets. Most tags have an opening tag and a closing tag. Some tags (called "void" tags) do not require a closing tag (e.g. br).
  • Element: refers to everything within a set of opening and closing tags.
  • Attribute: is a property of an HTML element, such as the use of a link reference.

CSS

Or Cascading Style Sheets is the code written to control the style of a page and HMTL elements. It is used for describing the look and formatting of a document written in a markup language. There are three ways of including CSS styling in a web page:

  • Head: write CSS in the head tag of your HTML;
  • Add link: link your HTML to a separate CSS file; and
  • Inline: write your style inline with your HTML.

★ The method Adding Link is normally the most advisable as it keeps the HTML neatly organized, facilitating reading and editing. You can literaly change the entire look of the page without touching the HTML text, when properly done. It also helps building leaner coding structures, which allows a page to be loaded faster.


A Good Structure Is Your Best Ally to Avoid Repetition

Structured And Faster Pages

Structured frameworks have been used by leaders in many industries to organize someone’s thinking, break a problem into manageable chunks and improve communication. It's not a surprise that HTML also leverages a standard framework to organize it's grammar. HTML uses the common tree-like diagram. Another way to think about this embeded structure is to think about boxes. A box inside a box, inside a box, and so on.

CSS also has its own logical structure: cascading. It can also be described as an inheritance mechanism in which properties are applied not only to a specified element, but also to its descendants. It relies on the tree framework, same hierarchy of HTML elements. This structure allows CSS code to be lean, and therefore pages to load a lot faster.

★ A good structure is what allows documents to be properly organized, minimizing erros, making it easier for others to read your code, and most of all, it minimizes lines of code, avoiding repetition and helping pages be loaded faster.

Tree-Like Diagram

This is the way to represent a hierarchical structure in a graphical form. It is also sometimes referred to as DOM or Document Object Model. More information can be found on Wikipedia.

Cascading

Cascading means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect. More information can be found on Wikipedia.

Indentation

It is best practice to use indentation on HTML code writing to facilitate reading and navigation within the code. It does not affect how the page looks, but it does make the code source a lot nicer looking.


A Bit More About Boxing

Sizing

HTML elements are boxes and each box has 4 components: content, padding, border, margin. Because there are so many components to each box, it can often be hard to get the size of a box just right. There are two techniques to help sizing issues:

  1. Set sizes in percentage rather than pixels.
  2. Set the box-sizing attribute to border-box for every element.

Different browsers work differently. Sometimes this causes different browsers to display the same code differently.

Positioning

DIVs are block elements (as opposed to inline), so by default they take up the entire width of a page. Adding the rule "display:flex" to the appropriate CSS will override this behaviour and let DIVs appear next to each other.
